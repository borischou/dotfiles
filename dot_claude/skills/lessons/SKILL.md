# Lessons Learned

记录和回顾经验教训，防止重复犯同样的错。

## 文件位置

每个项目：`tasks/lessons.md`（项目根目录下）

## 模式

### 模式 1: 记录（默认）

当被纠正、修完 bug、或踩到坑时，追加一条记录。

**触发场景**：
- 用户纠正了错误做法
- `/debug` 流程完成后（Step 7 自动触发）
- 用户主动说"记住这个"

**流程**：

1. 读取 `tasks/lessons.md`（不存在则创建）
2. 检查是否已有类似记录（避免重复）
3. 追加新记录到文件末尾

**记录格式**：
```markdown
### L{序号} - {YYYY-MM-DD}

- **问题**: 一句话描述发生了什么
- **根因**: 为什么会发生
- **修复**: 怎么修的
- **规则**: 下次怎么避免（写成可执行的规则）
```

**示例**：
```markdown
### L001 - 2026-03-02

- **问题**: Python 类型标注用了 `X | None` 语法导致 3.9 环境 import 失败
- **根因**: `X | None` 是 3.10+ 语法，项目要求 3.9+
- **修复**: 改为 `Optional[X]` from typing
- **规则**: Python 项目始终用 `Optional[X]` 和 `Union[X, Y]`，不用 `|` 语法
```

### 模式 2: 回顾

会话开始时或用户要求时，回顾已有教训。

**流程**：

1. 读取 `tasks/lessons.md`
2. 如果文件存在且有内容，展示摘要：
   - 总记录数
   - 最近 3 条
   - 与当前任务相关的记录（如果能判断）
3. 如果文件不存在，告知用户暂无历史教训

## 初始文件模板

首次创建 `tasks/lessons.md` 时使用：

```markdown
# Lessons Learned

项目经验教训记录，防止重复犯错。由 /lessons skill 维护。

---
```

## 规则

- 每条记录必须包含 **规则** 字段 — 这是最重要的部分，要具体可执行
- 不记录一次性的、不会再犯的错误（如 typo）
- 相似的教训合并更新，不重复添加
- 记录用中文
- `tasks/` 目录不存在时自动创建
