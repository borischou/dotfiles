# Boss直聘 额度机制详细记录

日期: 2026-03-05 实测

## 两种打招呼渠道

### 1. 推荐页 `/web/chat/recommend`
- iframe: `recommendFrame`
- 按钮: `button.btn.btn-greet`（"打招呼"）
- 点击即发送默认招呼语，无需手动输入
- **免费额度: 3次/天**
- 额度用完弹窗: `.dialog-wrap.active` 包含"今日主动沟通数已达上限"
- 弹窗内容推销 VIP 高效版 (30天 ~10元/天) 和聊天加油包

### 2. 搜索页 `/web/chat/search`
- iframe: `searchFrame`
- 按钮: `button.btn-getcontact`
- 有畅聊卡时显示"畅聊卡 N/N"，无卡时显示"联系Ta"
- 点击后弹出**职位选择对话框**（"选择该牛人开聊职位"），需选择职位后确认
- **免费额度: 1张畅聊卡/天**
- 成功后提示: "使用成功 已为您发起沟通并索要简历/微信/电话"
- 热搜牛人需消耗多次畅聊卡（如3次），免费用户够不上

## 免费 vs VIP 对比

| 权益 | 免费版 | VIP 高效版 (30天) |
|------|--------|------------------|
| 每日主动查看 | 20 个 | 120 个 |
| 每日沟通总量 | 3 个 | 60 个 |
| 在线竞招职位 | 0 个 | 2 个 |
| 搜索畅聊卡 | 1 张/天 | 含在沟通总量内 |
| 价格 | 免费 | ~300元/月 (~10元/天) |

## 弹窗处理要点
- 购买弹窗关闭: `.dialog-wrap.active .boss-popup__close`
- 职位选择确认: 先点击职位行 → 再点击"搜索畅聊卡(N/N)"按钮
- 成功弹窗有"查看对话"和"关闭"两个按钮

## 对 batch_greet.py 的影响
- 免费账号: 脚本最多打 3-4 个招呼就会触发购买弹窗停止
- VIP 账号: 脚本可以正常批量运行，每天上限 60 人
- 建议: settings.json 中 `message_per_day: 80` 应改为 60（匹配 VIP 上限）
